# Production Checklist

Use this checklist before each production rollout for image-workflow.

## Auth and roles

- Admin can log in and log out without a hard refresh.
- Staff can log in and log out without a hard refresh.
- Staff cannot see or open Settings, Monitoring, or Costs.
- Admin can open Settings, Monitoring, Costs, Jobs, Asset Library, and KPI Dashboard.
- Temporary password users are forced to change password before workflow actions.
- Unknown `/api/*` routes return JSON 404 instead of HTML.
- Admin-only routes return 401 without a session and 403 for staff.

## Generate and approve workflow

- Generate Hero succeeds on Render.
- Approve Hero saves the approval and export record.
- Generate Support succeeds from the approved hero anchor.
- Approve Support saves expected outputs.
- Generation polling continues to work with an authenticated session.
- Failed or stuck generation can be retried by admin.
- Retry export still works when an approved/generated source is available.

## Google Drive

- Google OAuth status is visible only to admin.
- Google Drive connect flow starts from Settings.
- OAuth callback succeeds only from a valid state generated by the server.
- Token is persisted in `public.integration_tokens`.
- No OAuth `token_json`, access token, refresh token, or client secret appears in API responses or logs.

## Monitoring and costs

- KPI Dashboard loads for active users.
- Monitoring loads for admin and shows stuck, failed, and recent system events.
- Costs loads for admin and all values are clearly labelled as estimated.
- Failed and retry generation cost impact appears in Costs.
- Staff cannot open Monitoring or Costs by direct hash navigation.

## API and logging safety

- `/api/health` returns only safe booleans/status, never secret values.
- `/api/supabase/config` returns only public Supabase browser config.
- Server logs do not include passwords, service role key, FAL key, OAuth token JSON, provider tokens, or raw prompts.
- API errors use `{ ok:false, code, error }` for hardened routes and fallback paths.

## Backup and operations

- Confirm Supabase automatic backups are enabled before large workflow changes.
- Keep Google Drive output folder ownership and permissions documented.
- Keep Render environment variables in the Render dashboard, not in the repo.
- Rotate temporary staff passwords after account creation.
- Review failed jobs and cost impact weekly from Monitoring and Costs.

## Deploy

- Run `node --check app.js`.
- Run `node --check server.mjs`.
- Run `git diff --check`.
- Confirm Render has required environment variables: Supabase URL, anon key, service role key, FAL key, Google OAuth config when Drive export is enabled.
- Smoke test desktop and mobile after deploy.

## Known limitations

- Cost values are configured estimates, not provider invoices.
- Google OAuth callback must remain public so Google can redirect back, but it is protected by a short-lived server state.
- Existing historical events may not have full cost metadata; the dashboard estimates cost from generation records where needed.
